# 緊急廣播設定（客戶版）

CX-1000 的「緊急模式」是全系統最高優先的強制廣播機制，一旦啟動就會蓋過背景音樂與一般廣播，且不受廣播逾時（time-out）自動切斷；本頁說明三種緊急廣播類型、音源怎麼準備，以及如何用外部系統的三組接點（警告／避難／解除）自動觸發與復歸。

!!! danger "危險（務必先讀）"
    CX-1000 **並非 EN54／消防認證產品**。這裡介紹的是一般／緊急廣播功能的整合應用，**不得取代法規要求的消防偵測與疏散系統**。涉及人員安全的法定疏散，仍須採用合格的消防設備。

---

## 一、優先權：數字越小，優先權越高

緊急廣播之所以能「壓過一切」，靠的是優先權。CX-1000 用 1–160 分級，數字越小越優先。

| 區間 | 用途 | 說明 |
|---|---|---|
| 41 – 60 | 緊急麥克風廣播 | 站台即時人聲，緊急帶最高 |
| 61 – 80 | 緊急訊息廣播 | 內建／預錄的 Warning、Evacuation |
| 81 – 120 | 呼叫、對講通話 | 一般帶 |
| 121 – 140 | All-clear 解除訊息、一般廣播 | 一般帶 |
| 141 – 160 | 背景音樂（BGM） | 最低 |

!!! note "注意"
    - 進入緊急模式後，**BGM 與一般廣播會被整個封鎖**，緊急廣播不受廣播逾時切斷。
    - **All-clear（解除）屬於一般帶（預設 125）**，不是緊急帶——它只是「狀況結束」的通知音，播完系統就回到正常。

---

## 二、三種緊急廣播類型

| 類型 | 音源 | 優先權 | 觸發方式 | 可否播外部即時音源？ |
|---|---|---|---|---|
| 緊急麥克風廣播 | 站台麥克風即時人聲 | 41–60 | 站台按鍵 | 僅限站台麥克風 |
| 緊急訊息廣播 | 內建／預錄檔 | 61–80 | 按鍵或控制輸入 | 否（預錄） |
| 緊急訊息樣式（pattern） | 預錄序列 | 61–80 | 按鍵或控制輸入 | 否（預錄） |

!!! warning "警告：外部線路音源進不了緊急帶"
    外部設備送進來的「線路（AUX／line）音源」**最高只能到一般帶（優先權 121）**，無法當成緊急級廣播。緊急帶只接受「站台麥克風」與「系統內建的預錄檔」。
    （以 RM1700 為例：input1 麥克風可作緊急，input2 的 AUX 線路不可。）

外部緊急系統要整合進 CX，走的是**第三種「緊急訊息樣式」＋接點觸發**——由外部乾接點觸發 SM1000 內播放預錄好的緊急語音。

## 三、音源：內建 vs. 自錄

!!! tip "建議：內建只有提示音，語音要自己錄"
    CX 內建的緊急音源**只有提示音（chime）**，例如緊急廣播開始／結束的 pre-chime／post-chime。**沒有內建的口語緊急訊息**。因此「警告／避難／解除」的語音需自行錄製成 WAV 上傳登錄，可用內建 chime 當開頭注意音、後接自錄語音。

自錄 WAV 的硬性規格如下（不符會登錄失敗或播放異常）：

| 項目 | 規格 |
|---|---|
| 格式 | WAV / PCM |
| 位元深度 | 16-bit |
| 聲道 | 單聲道（Mono） |
| 取樣率 | 16 / 32 / 48 kHz |
| 檔名 | 5–15 字元（含 `.wav`），僅限英數與 `! @ # $ % & - _`，**不可有空白** |
| 單檔大小 | ≤ 250 MB |
| 全系統合計 | ≤ 500 MB，最多 999 檔（緊急訊息最多 200 種） |

!!! warning "警告：開頭一定要墊靜音"
    每個 WAV 檔開頭**務必墊一小段靜音**，否則廣播啟動時第一個字會被切掉（吃字）。
    （登錄後在軟體裡的「顯示名稱」可另取，最多 32 字，與檔名規格無關。）

## 四、序列與樣式的概念

要讓「警告 →（延時或手動）→ 避難」自動接續，靠的是**緊急序列（Emergency sequence）**。

- 一條序列最多 **3 個 phase（階段）**，每個 phase 綁一個音源與逾時設定。
- Phase 的音源可選 **No sound（無音，預設）** 或 **Warning／Evacuation 用途**的內容；**All-clear 不放在序列裡**（見第六節）。
- 讓 Phase 1（Warning）循環播放，並把 **Time-out 設為 ON、時長設一個夠長的安全值**（例如 10–30 分）；如此一來 Phase 2（Evacuation）才會出現，且能靠「避難」接點手動提前跳階；接點若一直沒來，超過安全時間才自動前進。

**緊急訊息樣式（pattern）** = 序列 ＋ 廣播目的地 ＋（選配）連動的控制輸出群組。實際觸發與廣播的就是這個 pattern。

!!! note "選單路徑"
    - 序列：`Broadcast → Emergency sequence`
    - 樣式：`Broadcast → Emergency message broadcast pattern`
    - 音源用途：`Audio file → Audio content`（Usage 選 Warning／Evacuation／All-clear，這三項要 Emergency mode 開啟後才出現）

### 讓 CX 靜音（無音緊急）

序列 Phase 的音源設為 **No sound（無音）**，就能做出「**進入緊急模式但不出聲**」——常用於「外部系統接管廣播時，讓 CX 鎖住自己並保持安靜」。

- 建一條序列，**Phase 1 音源 = No sound、Time-out = OFF**（單一階段，維持到解除）。
- 用它建緊急樣式，目的地選要靜音的區。
- 事件綁定（AF1062 或 **CC1016** 控制輸入皆可）：`Emergency message broadcast pattern start`。
- 觸發後 CX **進入緊急模式（鎖住 BGM／一般廣播）＋ 內容無音** → 等於用緊急模式讓 CX 靜下來；**需 Emergency all-clear 才解除**。

!!! tip "只想「壓著才靜、放開就恢復」？改用輸出音量衰減"
    若不想讓系統進入緊急模式、只要單純把輸出靜音，改用 `Event activation → Control input →` **Output volume attenuation (Level)**，把（2）Output volume 設 **`–∞`**（全衰減＝靜音）、目的地選要靜音的輸出。接點壓著就靜、放開即恢復，且**連緊急廣播也會被一併壓下**。

## 五、用外部三接點觸發（警告／避難／解除）

最常見的整合情境：外部緊急系統提供三組**乾接點**，接進 CX 的 **AF1062 控制輸入**，分別觸發不同的緊急事件。（AF1062 有 10 個控制輸入，綁哪個緊急函式都沒有站台限制，三組綽綽有餘。）

| 外部接點 | AF1062 控制輸入 | CX 事件函式 | 效果 |
|---|---|---|---|
| 警告 | CI-1 | Emergency message broadcast **pattern start** | 啟動樣式，播 Phase 1 = Warning |
| 避難 | CI-2 | Emergency sequence **phase transition** | 序列前進到 Phase 2 = Evacuation |
| 解除 | CI-3 | Emergency **all-clear** | 播 All-clear ＋ 退出緊急模式 |

### 接線重點（AF1062 控制輸入）

AF1062 控制輸入是**無電壓 make（乾接點）**：開放時約 24V DC、短路約 5mA，負極 1–10 內部共通。

```
外部接點一腳 → CI 的「＋」（CI-1 / CI-2 / CI-3）
外部接點另一腳 → 任一「−」（共通）
三組可共用一條共通回線
```

!!! warning "警告：只能送乾接點，不可帶電"
    - 外部**必須是無電壓（乾）接點**，24V／5mA 由 AF1062 機內供給；外部只負責「短路／開路」，**不可灌入任何帶電訊號**，否則會誤讀或損壞。
    - **極性**：外部若是機械接點（繼電器／按鈕）不分正負，隨便接；若外部是**半導體輸出（開集極／SSR）就要對極性**——AF「＋」接電晶體集極（C）、AF「−」接射極（E／GND），接反不動作。不想煩惱極性，可在中間加一顆隔離繼電器，把半導體變回機械乾接點。
    - 接點閉合需 **≥ 0.2 秒**，CX 才會判讀。
    - 確認 AF1062 的 **24V DC 電源已接**。

### 設定步驟摘要

1. **先開緊急模式**：`General →（3）Emergency mode = Enable`。
   !!! danger "危險：這步一定要最先做"
       General 的項目一經更改會**重置相關設定**，且緊急函式／用途／優先權欄位都要在 Emergency mode 開啟後才出現。務必最先設定，再做其他步驟。
2. 上傳三個 WAV：`Audio file → Audio file registration`（User files 分頁 → Register）。
3. 組成三則 Audio content（Warning／Evacuation／All-clear），設好 Usage：`Audio file → Audio content`。
4. 建序列與樣式（第四節）。
5. 事件綁定：`Event activation → Control input`，Device 選 **CX-AF1062**，把 CI-1／CI-2／CI-3 分別綁上表三個事件函式。
6. 存檔與上傳：`File → Save`（存成 `cx1000settings.zip`，勿改名）→ `Communication → Settings & audio file upload (PC→SM)`，等狀態全部變綠；**上傳中請勿斷網、斷電**。

!!! note "注意：純接點觸發不必設定站台 Usage"
    只有想讓某台操作站「手動按鍵啟動／解除」或「講即時緊急廣播」時，才需把那台站台設成 `Usage = General/Emergency`。全靠 AF1062 接點觸發的這套整合**可跳過站台設定**；AF1062 本身也不在 Usage 清單裡，只要 Emergency mode 開著就能吃緊急函式。

## 六、All-clear 與退出緊急模式

!!! danger "危險：系統卡在緊急模式時，唯一出口就是 Emergency all-clear"
    緊急模式會鎖住 BGM 與一般廣播。**要停掉所有緊急訊息、退出緊急模式並播放解除訊息，唯一的方法就是觸發「Emergency all-clear」事件**（外部解除接點 → CI-3，或站台按鍵）。這一個事件會一次完成三件事：停止緊急訊息、退出緊急模式、播放 All-clear。若沒有配置 all-clear 觸發，系統將無從復歸。

因此請注意：

- All-clear **不是序列的一個 phase**，序列裡選不到它是正常的。它掛在 **`Emergency all-clear` 事件的詳細設定 →（1）Audio content**，復歸時才播。
- 一組接點就能跑完整套序列（序列會自動往下走），但 **all-clear 一定要另一個觸發**（另一接點或站台按鍵）。
- **解除範圍**：AF1062 的 all-clear 只清「它所屬的 hub」；多 hub 系統要確認這顆 AF1062 就在你要解除的 hub 內。

## 七、關於 IP-A1 喇叭（補充）

!!! warning "警告：緊急送不到走網路 multicast 的 IP-A1"
    CX 的緊急廣播**無法透過網路 multicast 送到 IP-A1**（IP-A1 的 multicast 目的地只掛在「Wide area」型樣式，而緊急目的地下拉不吃 Wide area）。
    可行的做法是**類比線路橋接**：把某個 AF1062 線路輸出實體接到 IP-A1 系統的音訊輸入，該輸出就會出現在緊急樣式的 `Individual → Broadcast audio output` 下拉裡，勾選它即可。IP-A1 側需自行把 line-in 廣播出去、優先權設得夠高。

---

## 重點整理

- **緊急模式 = 全系統最高優先的強制廣播**，會鎖住 BGM 與一般廣播，且不受廣播逾時切斷。
- **三種類型**：站台即時麥克風（41–60）、內建預錄訊息（61–80）、預錄序列樣式（61–80，可接點觸發）。外部整合走「序列樣式＋接點」。
- **可做「靜音緊急」**：序列 Phase 音源設 `No sound` ＝ 進緊急模式但不出聲（需 all-clear 解除）；或用 `Output volume attenuation (Level)` 設 `–∞` 做「壓著才靜、放開恢復」的純靜音。
- **外部線路音源進不了緊急帶**，最高只到一般 121；緊急語音要**自己錄 WAV**（16-bit／Mono／16·32·48 kHz、檔名 5–15 字元、開頭墊靜音）。
- **三接點對應**：警告 → CI-1（pattern start）、避難 → CI-2（phase transition）、解除 → CI-3（all-clear）；外部須為**無電壓乾接點**，閉合 ≥ 0.2 秒。
- **一定先開 Emergency mode 再做其他設定**；接點只能送乾接點、半導體輸出要對極性。
- **系統卡在緊急模式的唯一出口 = Emergency all-clear**（會同時停緊急、退出緊急模式、播 All-clear）。
- **非 EN54／非消防認證，不得取代法定消防疏散系統。**
