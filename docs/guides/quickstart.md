# 快速上手 / 系統初始化

CX-1000 的初始化，本質上是「在電腦的設定軟體裡把整套系統畫好、存檔，再上傳到 CX-SM1000 主機生效」。以下把整個流程拆成七個階段，並標出最容易踩雷的地方。

!!! note "注意"
    CX-1000 全系統的大腦是 **CX-SM1000 系統管理主機**。所有設定都先在 PC 端的「CX-1000 設定軟體」完成，最後上傳到 SM1000 才會真正生效——設定軟體本身不會即時改到現場設備。

## 流程總覽

| 階段 | 內容 | 在哪裡做 |
|---|---|---|
| 0 前置準備 | 下載軟體/韌體、規劃 IP、準備 PC | 辦公室 |
| 1 硬體安裝接線 | 機構固定、電源、網路 | 現場 |
| 2 設定軟體安裝 | 在 PC 安裝設定軟體 | PC |
| 3 建立設定檔 | 登入、General、裝置註冊、細項、存檔 | 設定軟體（離線） |
| 4 裝置網路設定 | 用偵測工具逐台設 IP | 設定軟體（連線） |
| 5 帳密同步 | 讓 SM1000 帳密與設定軟體一致 | 瀏覽器 |
| 6 上傳設定 | 把設定檔寫入 SM1000 | 設定軟體或 SD 卡 |
| 7 驗證啟用 | 自我檢查、功能實測 | Web 維護 + 現場 |

---

## 階段 0：前置準備

1. **閱讀安全注意事項**，確認接地與電源需求。
2. **下載軟體與韌體**：到 TOA DATA Library 搜尋「CX-SM1000」，取得設定軟體、各裝置最新韌體、相容 SD 卡清單與手冊。
3. **準備設定用 PC**：Windows 10（21H1 以上）/11 64 位元、Core i5-6200U 以上、4GB 以上記憶體、1920×1080 以上解析度，並需安裝 .NET 6 與 PostgreSQL 13。
4. **規劃 IP 配置表**：先把每台裝置的 IP、子網遮罩、閘道、Device ID（1–4000）列成一張表。

!!! danger "危險"
    **所有裝置出廠預設 IP 全部都是 `192.168.14.1`。** 同一網段上不可能同時存在兩台相同 IP，因此每一台都必須在階段 4 逐台改成唯一值，否則會互相衝突、偵測不到。

!!! tip "建議"
    Windows 使用者帳號名稱請只用英數字。含中文或特殊符號可能造成資料庫或路徑問題。

---

## 階段 1：硬體安裝與接線

1. **機構固定**：19 吋機架、桌上型或壁掛擇一。
2. **電源**：CX-SM1000 使用 AD-246 變壓器，或透過 LAN A 走 PoE（IEEE 802.3af）供電。擴音機開機前先把音量轉到最小並確實接地。
3. **網路接線**：使用 Cat5 以上纜線、100BASE-TX。
4. **插入 SD 卡（SM1000）**：若採網路上傳，此時可暫不插卡。
5. **全部接妥後再送電。**

!!! warning "警告"
    若要做 LAN A／LAN B 雙線備援，交換器**必須支援 RSTP**，否則會形成封包迴圈導致網路癱瘓。不需要備援時只接其中一條即可。

---

## 階段 2：設定軟體安裝

安裝檔分為兩種，依 PC 現況擇一：

| 版本 | 適用情況 |
|---|---|
| **Without PostgreSQL** | PC 已安裝 PostgreSQL 13 |
| **With PostgreSQL** | PC 尚未安裝，安裝時一併裝好資料庫 |

依安裝精靈完成即可，桌面會建立捷徑。

---

## 階段 3：建立設定檔（離線設定）★ 重點

這是整個初始化的核心。此階段完全離線進行——先在軟體裡把系統「畫」出來，最後存成設定檔。

### 步驟 1：啟動與登入

1. 開啟設定軟體，於啟動選單選 **Create New**，資料庫建立成功會顯示 `Database OK`。
2. 設定「設定軟體」的登入 ID 與密碼：

```text
登入 ID：7–32 碼
密碼  ：5–32 碼
可用符號：! @ # $ % & - _
```

!!! warning "警告"
    這組 ID／密碼**必須與 CX-SM1000 主機一致**，之後才能線上上傳（SM1000 出廠值為 `cx-1000 / guest`）。兩邊不一致是上傳失敗最常見的原因之一。

### 步驟 2：General 一般設定（務必最先定案！）

側邊選單原則是「由上往下設，只設需要的項目」。而 General 之所以要**最先**做，是因為它會牽動大量下游設定。

!!! danger "危險：General 一改就重置，且無法復原"
    除了「系統名稱」以外，**變更 General 裡任何一項，都會把相關設定重置回預設值，而且改回去也不會還原。** 因此請先把 General 全部決定好，再開始做細項設定。

General 需要決定的關鍵項目：

- **系統名稱**（預設 `CX-1000 System`，可隨時改）
- **故障偵測**：預設 Enable
- **緊急模式**：預設 Disable
- **Hub 功能**（預設 Disable）
- **Jitter buffer**（預設 Standard）

**Hub 功能怎麼選：**

| 情況 | 選擇 |
|---|---|
| 裝置數超過 1000 台 | **必須 Enable** |
| 跨多棟建築、想依棟分群 | 建議 Enable |
| 單一場所、未滿 1000 台 | Disable（較單純，建議） |

!!! danger "危險：Hub 的不可逆特性"
    一旦裝置數超過 1000 台，就**不能再從 Enable 改回 Disable**。若預期未來會擴充到接近或超過 1000 台，請一開始就設 Enable。
    （裝置計數 = 全部裝置扣掉 CX-SM1000 與攝影機。）

**Jitter buffer 怎麼選：**

- **Standard**：穩定、相容性佳，適合絕大多數案場（建議）。
- **Low latency**：延遲更低、反應更快，但可能造成 SIP 通話異常、視網路品質產生雜音、擴音機廣播受限。

!!! warning "警告"
    只要系統中註冊了任何一台 **CX-OP1500**，就**不能**選 Low latency。除非確定需要極低延遲、且無 OP1500、SIP 用量少、網路乾淨，否則一律用 Standard。

### 步驟 3：裝置註冊（Device Registration）

1. 先註冊 **System Manager（CX-SM1000）**，其 Device ID 預設為 `1`。
2. 其餘裝置不必一台台建立——按 **Add**，輸入各機型的「總數」後確定，系統會批次自動加入清單。Device ID 會自動指派最小未用號，名稱預設為「機型 + ID」（如 `CX-OP1700 24`），可事後再改。

裝置數量上限：

| 項目 | 上限 |
|---|---|
| 有 IP 的裝置總數 | Hub 關 ≤ 1000；Hub 開 ≤ 3000（皆不含攝影機） |
| SIP／外線介面／IP 攝影機 | 100／8／300 |

!!! warning "警告"
    即使開了 Hub，**一次註冊也不能超過 1000 台**（初始全歸 Hub No.1，每個 Hub 上限 1000 台），超過時需先分組。另外，裝置註冊越多，軟體操作速度會越慢。

### 步驟 4：其餘側邊選單（由上往下設）

註冊裝置 ≠ 設定 IP。此處只是在設定檔裡填入各裝置「應有」的 IP，實機 IP 要到階段 4 才寫入。設定順序建議由上往下，因為後面的項目會引用前面的設定。

| 類別 | 項目 | 說明 |
|---|---|---|
| **幾乎必設** | Network (Device) → IP address | 每台唯一 IP，須與階段 4 實際寫入值一致 |
| | Basic operation | 定義各機音訊進出與控制輸出 |
| | Broadcast 與／或 Intercom | 廣播系統設 Broadcast；有對講設 Intercom |
| | Security → ID/Password | 各裝置 Web 登入帳密，影響上傳對齊 |
| **用到才設** | Audio file、Record、Program timer、VOX | 沒用到就略過 |
| | SIP／IP camera／Video | 有相關設備才設 |
| **可先用預設** | Network (System) | 純 CX-1000 網路時免改；與其他系統共網才需調整 |
| | Priority、Audio scene、Lighting | 有預設，現場再微調 |

!!! tip "建議：兩個常被忽略但通常要設的項目"
    - **Event activation → Key**：Direct 鍵、廣播鍵要觸發什麼，要在這裡指派，否則按了沒反應。
    - **Basic operation → Device**：各機型的個別設定。

### 步驟 5：存檔

完成後存檔：

> 選單 → **File → Save as**

!!! note "注意"
    若之後要走 SD 卡上傳，請把設定檔存到 SD 卡的 `cx1000` 資料夾內。

---

## 階段 4：裝置網路設定（Device Detection Tool）

這一步把「設定檔裡規劃的 IP」實際寫進每一台設備。

1. 確認所有裝置已接網並送電（指示燈亮或閃）。
2. 開啟偵測工具：

    > 選單 → **Maintenance → Device detection & network settings**

    （PC 有多個 IP 時要先選來源 IP；防火牆需放行 `TOAFinder.exe`。）

3. 點 **Detect devices** 偵測。出廠裝置會顯示 IP `192.168.14.1`、遮罩 `255.255.255.0`、閘道 `0.0.0.0`。
4. 逐台修改 IP／遮罩／閘道／Web port／**控制埠（預設 15001）**，數值須與設定軟體一致。
5. 點 **Update settings** 寫入，裝置會自動重置。

!!! warning "警告"
    變更 **CX-SM1000 本身的 IP** 之後，全系統所有裝置都需要手動重新啟動。

!!! note "注意"
    跨路由器的裝置、SIP、攝影機、IP-A1 系列喇叭可能無法被自動偵測，需另行手動加入或用各自的工具設定。

---

## 階段 5：CX-SM1000 帳號密碼同步

1. 用瀏覽器登入 SM1000：

    ```text
    網址：http://192.168.14.1   （或改過的新 IP）
    出廠帳密：cx-1000 / guest
    ```

2. 進入設定並修改帳密：

    > 選單 → **Maintenance → Login ID**（需 Administrator 權限）

    將 ID／密碼改成與「階段 3 步驟 1」設定軟體所設的完全一致（ID 7–32 碼、密碼 5–32 碼）。

!!! tip "建議"
    這一步就是為了讓兩邊帳密對齊。上傳前務必再次確認兩邊相同，可省去大量除錯時間。

---

## 階段 6：上傳設定檔

上傳前請先逐項確認：

!!! warning "警告：上傳前檢查清單"
    - [ ] 設定軟體與 SM1000 的 ID／密碼已對齊
    - [ ] 各裝置 IP 與控制埠（15001）已設定完成
    - [ ] 設定檔已存檔
    - [ ] 所有裝置已連網
    - [ ] SM1000 狀態燈為綠色
    - [ ] 系統非緊急模式、無進行中的通話或廣播

**方式 A：網路上傳（建議）**

> 選單 → **Communication → Settings & audio file upload (PC→SM)** → 選資料夾 → 上傳

進度指示全部亮綠即代表成功。

**方式 B：SD 卡上傳**

1. 將設定檔存到 SD 卡的 `cx1000` 資料夾。
2. 插入 SM1000。
3. 將 **Setting switch 開關 2** 由 OFF 撥到 ON（燈由閃爍轉為恆亮即開始上傳）。
4. 恆亮後約 1 分鐘完成，將開關 2 撥回 OFF，等 30 秒再取卡。（開關 1 為下載用）

!!! danger "危險：上傳中絕對不要斷電或斷網"
    無論網路或 SD 卡方式，上傳過程中**都不可以關機、拔線、斷電或取出 SD 卡**。中途中斷可能導致設定寫入不完整，甚至需要重來。

!!! note "注意：上傳失敗（紅燈）怎麼看"
    - 紅燈停在**前段** → 通常是 PC 與 SM1000 之間的問題（帳密、IP、緊急模式、版本不符）。
    - 紅燈停在 **Applying settings to each device** → 通常是 SM1000 與各裝置之間的問題（某台裝置 IP 或控制埠設錯）。

---

## 階段 7：驗證與啟用

1. 用 Web 維護登入，確認運作狀態、日誌與日期時間（可設 NTP 自動校時）。
2. 執行**自我檢查**，確認各裝置與線路無故障。
3. 功能實測：廣播、呼叫／對話、緊急廣播（若啟用）、門禁、定時器。
4. 日後若透過 Web 維護改過設定，記得用設定軟體連線**下載**設定檔回 PC 保存，避免版本不一致。

---

## 重點整理

- **順序決定一切**：先定案 General，再做細項。General 除系統名稱外，一改就把相關設定重置且不可復原。
- **Hub 與 Jitter 是「一次性決定」**：超過 1000 台必須開 Hub 且之後不可關回；只要有一台 OP1500 就不能用 Low latency。
- **IP 一定要逐台改**：所有裝置出廠都是 `192.168.14.1`，靠 Device Detection Tool 改成唯一值並寫入。
- **帳密要對齊**：設定軟體與 SM1000（出廠 `cx-1000 / guest`）帳密一致，才能網路上傳。
- **上傳中勿斷電斷網**：全綠為成功，紅燈依停留位置判斷是「PC↔SM」或「SM↔裝置」問題。

### 關鍵預設值速查

| 項目 | 值 |
|---|---|
| 裝置出廠 IP | `192.168.14.1` |
| 子網遮罩／閘道 | `255.255.255.0` / `0.0.0.0` |
| 控制埠 | `15001` |
| Web 登入 | `http://192.168.14.1`，`cx-1000 / guest` |
| 設定軟體 ID／密碼 | 7–32 / 5–32 碼（須與 SM1000 一致） |
| Device ID | 1–4000（不可重複，SM1000 預設 1） |
| 裝置上限 | Hub 關 ≤ 1000、Hub 開 ≤ 3000（不含攝影機） |
| 網路 | 100BASE-TX、Cat5 以上；備援需 RSTP；LAN A 支援 PoE（802.3af） |
| SD 卡開關 | 開關 2 = 上傳、開關 1 = 下載 |
