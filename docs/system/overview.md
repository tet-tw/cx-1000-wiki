# CX-1000 系統架構與概念

CX-1000 是 TOA 的 IP 網路通訊系統，把「影像對講」與「多路一般廣播」整合在同一套乙太網路架構上，並可再擴充緊急廣播應用。

本頁幫助你第一次認識這套系統：它是什麼、能做什麼、各設備如何用網路串起來，以及大腦 CX-SM1000 與 IP 喇叭在其中扮演的角色。

---

## CX-1000 是什麼

CX-1000 是一套以 **IP 網路（乙太網路）** 為骨幹的通訊與廣播平台。傳統上，門口對講、辦公室對講、背景音樂、一般廣播往往是各自獨立的系統；CX-1000 把這些整合到同一張網路上，用一台系統管理主機統一管理，因此擴充、佈線與維護都更單純。

系統規模可以很小（幾台門口機加一台主機），也可以很大：

- 整套系統最多可容納 **3000 台裝置**
- 最多 **3000 路喇叭線路輸出**
- 支援 **SIP** 通訊協定，可與一般 SIP 話機互通
- 支援 **ONVIF®** 網路攝影機，可在影像操作台上監看畫面

!!! note "注意"
    CX-1000 是「通訊／廣播」系統，**並非** EN54／消防認證產品，不可用來取代法規要求的火警偵測或緊急疏散系統。緊急廣播功能屬於加值應用，仍需搭配合格的消防系統。

---

## 三大功能

CX-1000 的設備可粗分為「對講呼叫」與「廣播控制」兩類，組合出三種主要用途：

| 功能 | 說明 | 主要設備 |
|---|---|---|
| **對講／呼叫** | 站台之間雙向通話；影像站台可在通話時顯示對方攝影機畫面 | OP1700、CL1750、OP1500、CL1550 |
| **一般廣播** | 分區或全區廣播、背景音樂（BGM）、外部音源播放 | RM1700、AF1062、PA1120／PA1240、對講站台亦可作為廣播目的地 |
| **緊急廣播** | 站台即時緊急麥克風、內建預錄緊急訊息、可接點觸發的緊急序列 | 具緊急權限的麥克風站台＋主機內建預錄檔 |

!!! tip "建議"
    緊急廣播有獨立的優先權與行為規則（會鎖住 BGM 與一般廣播、不受廣播逾時切斷等），細節請參考「緊急廣播」專頁，本頁只介紹架構層面。

## 網路架構：乙太網路 + Multicast

所有 CX-1000 設備都透過 **100BASE-TX 乙太網路（RJ45）** 互連，建議使用 Cat5 以上網路線。

### 供電與備援

- 各站台的 **LAN A 支援 PoE（802.3af）**，接到支援 PoE 的交換器即可免用變壓器供電。
- 主機與多數設備具備 **LAN A／LAN B 雙埠**，可做網路備援；備援拓樸需搭配支援 **RSTP** 的交換器。

### 為什麼用 Multicast

一次廣播往往要同時送到很多台喇叭。若用「一對一」方式逐台傳送，會佔用大量頻寬。CX-1000 改用 **multicast（多播）**：音訊只送出一份，凡是加入該群組的裝置就會同時收到，效率高、延遲低。

系統中的 multicast 位址、群組與埠號都在設定軟體中規劃：

!!! note "選單路徑（CX-1000 設定軟體）"
    - 選單 → **Network (System) → Broadcast/call**：設定廣播／呼叫使用的 multicast 位址與埠號
    - 選單 → **Network (System) → Multicast group**：設定各裝置所屬的 multicast 廣播群組
    - 選單 → **Network (System) → IP-A1 multicast address**：設定廣播給 IP-A1 系列喇叭的參數

裝置出廠預設 IP 為 `192.168.14.1`、子網遮罩 `255.255.255.0`；實際部署時會在設定軟體中為每台裝置指定正式 IP。

!!! warning "警告"
    Multicast 廣播會在網段內擴散。若與其他系統共用網路，交換器建議開啟 **IGMP Snooping** 等機制管理群組流量，避免多播封包灌爆一般埠口。網路規劃請與資訊部門或系統整合商確認。

## CX-SM1000：系統的大腦

**CX-SM1000 系統管理主機（System Manager）** 是整套系統的核心，**每一套 CX-1000 系統都必須且只能有一台**。它負責：

- **儲存並執行整套設定**：所有裝置、優先權、廣播分區、事件動作都集中在此
- **統籌系統運作**：協調對講建立、廣播路由、緊急模式切換
- **故障偵測**：監看各裝置狀態，異常時發出警示（可由前面板按鍵確認／復位）
- **內建 Web 維護伺服器**：可用瀏覽器登入查看狀態與日誌

設定流程是「離線編輯 → 上傳生效」：先在電腦上用 **CX-1000 設定軟體** 建立設定檔，再上傳到 CX-SM1000，主機套用後全系統才會依新設定運作。

!!! danger "危險"
    在系統運作中按下 SM1000 的 Reset 重啟鍵，會使系統在重啟期間**全部停止運作**。請務必在確認無廣播／通話需求時才進行。

## Hub 分區概念（邏輯分區）

當系統規模較大時，可啟用 **Hub 功能** 把裝置分組管理：

- **未啟用 Hub**：整套系統最多 **1000 台裝置**（不含攝影機），視為單一群組。
- **啟用 Hub**：最多可設 **100 個 Hub**，每台裝置歸屬其中一個 Hub，**每個 Hub 最多 1000 台**，全系統可擴充到 **3000 台**。

啟用 Hub 後，可依「棟別／區域」獨立管理，例如一座由多棟建築組成的廠區，每棟設為一個 Hub：

- **管理權限（Administrator／Maintenance）** 可檢視與操作所有 Hub。
- **一般權限（User）** 只能檢視／操作被授權的 Hub。
- **緊急模式可依 Hub 為單位** 個別啟動與解除，不必牽動整個系統。

!!! warning "警告"
    這裡的「Hub」是**設定軟體中的邏輯分區（管理群組）概念，不是實體網路交換器**。名稱雖同，意義不同，規劃時請勿混淆。啟用位置：選單 → **General → Hub function → Enable**；分區歸屬：選單 → **Hub → Hub／Hub affiliation**。

## 與 IP-A1 系列 IP 喇叭的關係

除了 CX 自家的喇叭選擇器＋擴大機，CX-1000 也可把音訊送到 **TOA IP-A1 系列**的**接收端（播放）裝置** —— 主要是 **IP-A1SC15（IP 號角喇叭）** 與 **IP-A1AF（IP 音訊介面，輸出到擴大機／喇叭）**。這些接收端**透過網路 multicast 直接接收 CX 送出的廣播音訊**，適合以純網路方式延伸廣播覆蓋範圍。

!!! note "IP-A1 分「接收端」與「送話端」，別搞混"
    IP-A1 產品線分兩類，方向相反：

    - **接收端（播放）**：如 **IP-A1SC15**（IP 喇叭）、**IP-A1AF**（音訊介面）—— 會收 multicast 並播出，**這才是 CX「送音訊過去」的目的地**。
    - **送話／來源端**：如 **IP-A1RM**（遠端麥克風）、**IP-A1PG**（呼叫閘道器）—— 是把聲音「**送進**」IP-A1 網路的裝置，**本身不是播放端**，CX 不會（也無法）把廣播送給它們去「播」。

    規劃「CX → IP-A1」時，目的地務必挑**接收端**。

!!! warning "警告"
    **一般廣播** 送 IP-A1 走 multicast 可行；但 **緊急廣播** 送 IP-A1 走 multicast 不可行（緊急目的地不接受 Wide-area 型裝置）。若需將緊急廣播送到 IP-A1，必須改用**類比線路橋接**：AF1062 的 line out 實體接到 IP-A1 的音訊輸入，再於 Individual 目的地下拉選取。細節請見「緊急廣播」與「音訊橋接」專頁。

## 與 IP 網路攝影機整合

CX-1000／IP-A1 也能與 **IP 網路攝影機**（例如我們採用的 **TRIFORA N-C6 系列**）整合：攝影機偵測到畫面事件後，可用 **HTTP 通知**或**乾接點**觸發廣播（如闖入警示、導引語音）。此外，CX 的影像操作台本身也支援 **ONVIF®** 攝影機即時監看。

詳細做法與**實測 SOP**見 [整合方案 → 攝影機事件觸發](../integrations/camera.md)。

| **CX-OP1700** | 影像操作台 | 觸控 LCD、通話顯示攝影機畫面、可控制廣播與外部裝置 | 100 |
| **CX-CL1750** | 影像門口機 | 內建攝影機的門口機，畫面顯示於影像操作台 | 3000 |

| **CX-CL1550** | 門口機 | 標準門口機 | 3000 |
| **CX-RM1700** | 觸控遠端麥克風 | 觸控 LCD 遠端廣播麥克風（可作緊急麥克風） | 100 |
| **CX-AF1062** | 音訊介面 6in2out | 6 音訊入／2 音訊出，接外部音源或擴大機；含控制 I/O | 300 |
| **CX-PA1120** | 擴大機 120W 10SS | 內建 120W 擴大機＋10 分區喇叭選擇器 | 300 |

整套系統（含所有裝置）合計可達：**3000 音訊輸入、3000 音訊輸出、12000 控制輸入、15000 控制輸出、3000 喇叭線路輸出**，但不得超過各機型的單機連接上限。

!!! note "注意"
    即使未超過各項最大數量，某些**裝置組合**仍可能無法成立。實際可行的配置請以設定軟體驗證為準，並在規劃階段與 TOA 代理商確認。

## 重點整理

- CX-1000 是以乙太網路為骨幹的 IP 通訊／廣播平台，整合**對講、一般廣播、緊急廣播**三大用途。
- **CX-SM1000 是唯一且必備的大腦**，集中儲存設定、執行系統、偵測故障；設定採「離線編輯 → 上傳生效」。
- 設備以 **100BASE-TX** 互連，LAN A 支援 **PoE（802.3af）**，可用 LAN A／B 搭配 RSTP 做備援。
- 廣播以 **multicast** 一對多傳送，效率高；multicast 位址與群組在設定軟體中規劃，共用網路時建議開 IGMP Snooping。
- **Hub 功能是邏輯分區（非實體交換器）**：啟用後最多 100 個 Hub、全系統可達 3000 台，可依區域分權管理、分區操作緊急模式。
- **IP-A1 IP 喇叭** 走 multicast 接收一般廣播；但**緊急廣播送 IP-A1 需改用 AF1062 類比線路橋接**。
- CX-1000 **非消防認證**，不可取代法規要求的火警偵測／疏散系統。
