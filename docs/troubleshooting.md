# 故障排除

系統出現異常時，先照這頁的流程判讀與處理：**看燈號 → 查日誌 → 對症狀表**。

!!! note "✅ 原廠標準操作"
    本頁皆為 CX-1000 原廠內建的故障偵測、指示與日誌功能，屬 TOA 原廠手冊涵蓋範圍。

!!! warning "前提：故障偵測功能要先啟用"
    故障通知、Self-check、維護檢視都要在 `General` 啟用 **Failure detection** 才會運作。**若功能被停用，維護畫面與 Self-check 都看不到** —— 排查前先確認它是開的。

---

## 一、先做這三步

1. **看燈號** —— 判斷哪一台故障、是否已被確認（見第二節）。
2. **查日誌** —— 瀏覽器登入 CX-SM1000 → **Device monitor**（故障裝置標紅）或 **Failure log**（見第四節）。
3. **對症狀表** —— 依現象找解法（見第五節）。

---

## 二、燈號判讀

### Status 指示燈（各裝置共通）

| 狀態 | 意義 |
|---|---|
| 不亮 | 電源關／啟動中／尚未連上 System Manager |
| **綠燈亮** | 正常（已與 SM1000 建立連線） |
| 綠燈快閃（0.1 秒） | 收到呼叫（站台）／存取 SD 卡（SM1000） |
| **🟡 黃燈閃（1 秒週期）** | **本機故障，且「尚未確認」** |
| **🟡 黃燈恆亮** | **本機故障，且「已確認」** |
| 🔴 紅燈亮 | 處於緊急模式（不是故障） |

!!! tip "快速判讀"
    **黃燈就是故障**：閃＝還沒人按確認、恆亮＝已確認但問題還沒排除。
    **CX-SM1000 的指示燈**：系統內**任一**裝置故障就會亮 —— 它是全系統的總指示。

### RUN 指示燈（綠）

| 狀態 | 意義 |
|---|---|
| 不亮 | 電源關（或 PA1120／PA1240 切到 CPU OFF） |
| 恆亮綠 | 所有功能可用 |
| 閃綠（亮 1 秒／滅 1 秒） | 開機後功能尚未就緒的過渡狀態 |

### 其他指示

- **訊號指示燈**（AF1062／PA1120／PA1240）：音訊**輸入紅燈＝輸入過大**；**輸出紅燈＝超過額定輸出位準**。
- **LCD Failure 圖示**（OP1700／RM1700）：待機畫面上出現，故障排除後自動消失。

## 三、故障處置流程

```text
蜂鳴器響 + 黃燈閃爍
   ↓  按 Fault ACK 鍵（故障確認）
蜂鳴器停、黃燈轉為恆亮
   ↓  查日誌找出原因 → 排除
   ↓  按 Fault reset 鍵（故障重置）
指示熄滅、恢復正常
```

- **CX-SM1000 的 Fault ACK 鍵**（作為控制輸入時）可**一次確認全系統**所有故障輸出樣式；用其他按鍵則只確認該鍵所指派的那一組。
- 蜂鳴器可用 **CX-SM1000 機身 Setting switch 3** 停用（出廠為啟用）。

---

## 四、用日誌查原因（瀏覽器登入 CX-SM1000）

| 工具 | 用途 |
|---|---|
| **Device monitor** | 列出所有裝置狀態，**故障標紅**；可按 Reset failure status 清除 |
| **Failure log（故障日誌）** | 偵測到的所有故障，含**發生／確認時間**；可篩選、下載文字檔 |
| **Operation log** | 操作、事件、狀態變化與**操作失敗**紀錄 |
| **Stream log（串流日誌）** | 廣播／通話的串流紀錄，**診斷網路壅塞**用 |

!!! note "存取限制與容量"
    - **Stream log 只有設定軟體 `Security → ID/Password` 的 No.1 號 ID 看得到**。
    - 日誌容量：操作＋故障合計最多 100,000 筆、串流最多 10,000 筆；超量會存到檔案伺服器的 logs 資料夾。

## 五、症狀對照表

| 症狀 | 檢查與解法 |
|---|---|
| **機器完全不動作** | 電源是否接妥（**部分機型不支援 PoE**，須用指定 AC 變壓器）；PA1120／PA1240 的 **CPU OFF 開關**是否被切掉 → 撥回原位 |
| **站台不動作** | PoE 交換器**供電是否足夠**（要能供應所有站台）；是否使用指定的 AC 變壓器 |
| **設定軟體偵測不到裝置** | 所有裝置與 PC 必須在**同一 LAN**（跨路由器偵測不到）；PC 防火牆放行裝置搜尋程式；暫時關閉防毒軟體 |
| **設定檔上傳失敗** | 看**紅燈停在哪一步**：<br>・停在前段（Server state～Applying system settings）＝ **PC ↔ SM1000** 問題 → 檢查 ID／密碼是否與 SM1000 一致、IP／子網／閘道、SM1000 是否亮綠燈、**是否在緊急模式（緊急模式無法上傳）**、軟體與韌體版本、Proxy／防火牆<br>・停在 **Applying settings to each device** ＝ **SM1000 ↔ 各裝置** → 用裝置偵測工具確認每台的 IP／子網／閘道／**控制埠（15001）** 都正確 |
| **上傳很慢** | 大量音檔上傳每台可能超過 15 分鐘 —— **是慢不是壞，切勿中途斷線** |
| **瀏覽器開不了維護畫面** | PC 的 IP／子網／閘道是否正確；瀏覽器若走 Proxy 可能顯示異常 → 洽網管 |
| **無聲音／聲音失真** | 話筒、耳機、擴大機／喇叭接線；音訊輸入輸出音量與**輸入位準選擇開關**；壓縮方式與取樣頻率；麥克風開口是否被擋；**multicast 資料若過不了路由器 → 改用單播或洽網管**；⚠ **Emergency 控制輸入端子沒接好會把訊號衰減到 −∞** |
| **喀嗒雜音／音訊中斷** | 網路壅塞 → **查 Stream log** 確認後洽網管 |
| **站台話筒無聲** | 話筒是否正確接到站台 |
| **SIP 話機／VoIP 轉接器不通** | SIP ID／密碼／SIP 伺服器（＝SM1000）是否設對；**codec 僅支援 G.722 與 G.711** |
| **按 Direct 鍵沒反應** | 設定是否已**上傳**；觸控時勿碰到按鍵以外區域；是否確實按到 |
| **控制輸出沒作用** | 設定軟體是否設好控制輸出功能並**上傳** |
| **撥號時發生錯誤** | 各裝置註冊（型號／IP／站號）是否正確；一鍵撥號是否也登錄了號碼；**若正在用內建音源廣播，等廣播結束再操作** |
| **無法接收呼叫** | 等狀態燈亮起；仍不亮 → 確認站號設定正確 |
| **視訊無法建立** | 是否使用支援視訊的站台；系統是否已在設定軟體設為視訊用並上傳 |
| **無攝影機影像** | 鏡頭前是否被遮擋 → 移除遮蔽物 |
| **LAN／LINK 燈不亮** | 線材與埠位是否正確；需 **100BASE-TX 相容**交換器；交換器電源是否開啟 |
| **裝置停止運作＋顯示異常溫度** | 是否靠近熱源或日曬、是否長時間視訊／接收 IP 攝影機影像 → 遠離熱源、**暫停視訊或攝影機接收**讓裝置降溫 |

## 六、自我檢查（Self-check）

- **排程檢查**：每天定時自動執行。**手動檢查**：可隨時對選定的 hub／項目啟動。
- **可偵測項目**：免持 MIC／SP 音訊線路、**呼叫鍵是否卡住**、**擴大機（僅手動檢查可測）**、外部儲存、**LAN port A／B**、**Composition（設定與實機組態是否一致）**、Internal（DC 電壓／CPU 溫度）等。
- **結果判讀**：OP1700／RM1700 以圖示顯示；OP1500 以文字顯示 `OK`／`RUN`（檢查中）／`SKIP`（被通話或廣播略過）／**`FAIL`（異常）**。

!!! warning "檢查時的注意事項"
    - **檢查進行中請勿按任何鍵**，否則 MIC／SP 可能被誤判為故障，必須重做。
    - 附近若有其他站台同時檢查，麥克風可能收到別台的檢查音而誤判。

## 七、容易踩的雷

!!! danger "整群裝置同時失聯 → 先查「故障偵測主控機」"
    每 31 台裝置會指定一台 **Failure detection leader（故障偵測主控機）**。**這台若自己故障而與 SM1000 失聯，它群組下的所有機器會一起失聯停擺。** 整群同時掛掉時，優先檢查這台。

- **DC 電源**：輸入須在 **21.6～26.4 V DC**、容量 400 mA 以上，超出範圍會故障。
- **備援 LAN**：LAN A／B 接的交換器**必須相容 RSTP**，否則會產生封包迴圈。
- **散熱**：PA1120／PA1240 勿堵住散熱孔、需保持間距。
- **聲學回授**：免持通話環境改變可能嘯叫，通常續談會自動回復；否則調整音量或 voice switch。
- **勿在通話／廣播／事件進行中變更系統日期時間**，否則系統可能異常。
- **Web 設定頁**：按下確認前勿換頁、重載或關閉瀏覽器，否則設定會回到變更前。

## 重點整理

- **診斷三步**：看燈號 → 查日誌（Device monitor／Failure log）→ 對症狀表。
- **黃燈＝故障**（閃＝未確認、恆亮＝已確認）；**紅燈＝緊急模式**，不是故障。
- **處置流程**：Fault ACK（停蜂鳴）→ 排除原因 → Fault reset（復歸）。
- **網路類雜音／中斷**查 **Stream log**；**上傳失敗**看紅燈停在哪一步判斷是 PC↔SM1000 還是 SM1000↔裝置。
- **整群裝置同時失聯**，先查該群的**故障偵測主控機**。
- 故障偵測功能**停用時看不到維護檢視與 Self-check**，排查前先確認已啟用。
