# 整合方案 · 總覽

CX-1000 常見的對外整合做法整理如下。每一種都對應「**音訊怎麼進來 / 出去**」加「**用什麼觸發、走哪個優先權**」兩件事。

| 整合 | 用途 | 進 CX 的方式 | 說明頁 |
|---|---|---|---|
| **攝影機事件觸發** | 攝影機偵測到事件 → 自動廣播 | HTTP 通知 / 控制輸入 | [前往](camera.md) |
| **外部緊急系統（3 接點）** | 外部警報的 警告/避難/解除 → CX 內建緊急訊息 | 乾接點 → AF1062 控制輸入 | [前往](external-emergency.md) |
| **外部語音 / 校園廣播橋接** | 中控/校園系統廣播時 CX 一併輸出 | 音訊進 AF1062 + 開集極觸發 | [前往](audio-bridge.md) |
| **WebAPI** | 用網路方式觸發 CX | HTTP（Digest 認證） | [前往](webapi.md) |

!!! tip "選型原則"
    - 要**最高優先權、能蓋過一切**（真緊急）→ 走「外部緊急系統（3 接點）→ CX 內建緊急訊息」。
    - 只是**日常廣播 / BGM**（不需 override）→ 走「外部語音橋接」的一般廣播路徑。
    - 外部**線路音源無法**進到 CX 的緊急頻段（最高只到一般 121）—— 詳見各頁說明。

!!! warning "非 EN54"
    以上皆為 CX 一般／緊急廣播功能的整合應用，**非消防認證**，不得取代法規要求的消防系統。
