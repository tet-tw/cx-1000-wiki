# 整合方案 · 總覽

CX-1000 常見的對外整合做法整理如下。每一種都對應「**音訊怎麼進來 / 出去**」加「**用什麼觸發、走哪個優先權**」兩件事。

!!! tetplus "先分清楚：原廠標準 vs tet-tw 自研加值"
    本站整合方案分三類，頁面上會清楚標示，避免誤解：

    - ✅ **原廠標準功能** —— CX-1000／IP-A1 設備本身、TOA 原廠手冊涵蓋的功能（如乾接點觸發、AF1062 音訊橋接、CX 廣播／緊急樣式）。
    - 🔧 **tet-tw 自研加值** —— 由 **tet-tw 台灣分公司自行研發**的軟體／工具（如 **IP-A1 中控軟體**）所實現的整合。**非 TOA 原廠標準產品、不在原廠手冊範圍、非原廠背書**；可整合，但屬本公司加值服務（也可由客戶自備等效軟體）。
    - ⚙️ **原廠功能但有門檻** —— 屬原廠設計、但需額外條件（如 IP-A1 Remote API 指令規格**需向 TOA 書面申請**、CX WebAPI **公開零售韌體不一定內建**）。

    不確定某項屬於哪一類時，歡迎洽詢我們釐清。

| 整合 | 用途 | 進 CX 的方式 | 類別 | 說明頁 |
|---|---|---|---|---|
| **攝影機事件觸發** | 攝影機偵測到事件 → 自動廣播 | HTTP 通知 / 控制輸入 | ✅ 原廠(接點) ＋ 🔧 加值(軟體) | [前往](camera.md) |
| **外部緊急系統（3 接點）** | 外部警報的 警告/避難/解除 → CX 內建緊急訊息 | 乾接點 → AF1062 控制輸入 | ✅ 原廠標準 | [前往](external-emergency.md) |
| **外部語音 / 校園廣播橋接** | 現場既有的外部廣播主機廣播時，CX 一併輸出 | 音訊進 AF1062 + 開集極觸發 | ✅ 原廠標準 | [前往](audio-bridge.md) |
| **WebAPI** | 用網路方式觸發 CX | HTTP（Digest 認證） | ⚙️ 原廠但零售未必內建 | [前往](webapi.md) |
| **SIP 電話整合** | SIP 話機／PBX 與 CX 站台互打 | SIP（SM1000 內建伺服器） | ✅ 原廠標準 | [前往](sip.md) |

!!! tip "選型原則"
    - 要**最高優先權、能蓋過一切**（真緊急）→ 走「外部緊急系統（3 接點）→ CX 內建緊急訊息」。
    - 只是**日常廣播 / BGM**（不需 override）→ 走「外部語音橋接」的一般廣播路徑。
    - 外部**線路音源無法**進到 CX 的緊急頻段（最高只到一般 121）—— 詳見各頁說明。

!!! warning "非 EN54"
    以上皆為 CX 一般／緊急廣播功能的整合應用，**非消防認證**，不得取代法規要求的消防系統。
