# 外部語音 / 校園廣播橋接

把外部中控主機或校園廣播的「音訊＋控制訊號」接進 CX-1000 的 AF1062 音訊介面，就能讓中控在廣播時，CX 這一側的區域同步跟著播出；但要注意外部線路音源永遠進不了「緊急級」，而 CX 的緊急廣播要送到 IP-A1 喇叭必須走類比橋接、不能走網路 multicast。

!!! note "這頁在解決什麼"
    許多校園/廠區現場同時存在兩套系統：既有的**中控/校園廣播主機**，以及新導入的 **CX-1000**。常見需求是「中控一開麥克風/放公告，CX 負責的教室或戶外區也要一起響」。本頁說明如何用 **AF1062** 把兩套系統橋接起來，並釐清緊急廣播的能與不能。

---

## 一、讓 CX 隨中控「同步播出」（音訊＋開集極控制）

概念是：中控的**聲音**走一條線進 AF1062，中控的**開集極控制訊號**走另一條線進 AF1062；CX 用這個控制訊號當開關，決定「中控在播的時候，CX 這邊也一起播、放開就停」。

### 1. 硬體接線

| 中控端 | 線材 | CX / AF1062 端 |
|---|---|---|
| 線路輸出 Line out | 音訊線 | AF1062 音訊輸入（Audio input，共 6 組可選） |
| 開集極輸出 Control out | 控制線 | AF1062 控制輸入（Control input，共 10 組可選） |

!!! warning "開集極極性一定要對（接反不會動作）"
    AF1062 的控制輸入是「無電壓乾接點」介面，內部有一顆光耦合器，電流方向單向。若中控的控制輸出是**半導體/開集極（電晶體）**，就必須對極性：

    ```text
    AF1062 「＋」  →  中控電晶體集極（Collector，高側）
    AF1062 「−」  →  C 共通 / GND
    ```

    接反了電流走不通，CX 完全不會被觸發。若中控輸出是**機械繼電器乾接點**，則不分極性、正負隨便接；想省事也可以在中間加一顆隔離繼電器，把半導體輸出還原成機械乾接點。

### 2. 軟體設定（TOA 設定軟體）

!!! tip "設定順序：先設音源 → 建廣播樣式 → 綁控制輸入事件"

**步驟 1：登錄音訊輸入**

選單 → `Basic operation → Audio input`，把接中控的那組 AF1062 音訊輸入，**Usage（用途）設為 General（一般）**。

!!! danger "外部線路音源不能設成緊急"
    這裡只能是 General。外部「線路」音源在 CX 架構上**永遠進不了緊急頻段**，詳見下方第二節。

**步驟 2：建立一般廣播樣式**

選單 → `Broadcast → General broadcast pattern`，新增一個樣式：

| 欄位 | 設定 |
|---|---|
| 音源類型 Source type | Audio input（音訊輸入） |
| 音源 Source | 步驟 1 登錄的那組 AF1062 音訊輸入 |
| 目的地 Destination | 要跟播的區域（教室、戶外…） |

**步驟 3：把控制輸入綁到這個樣式**

選單 → `Event activation → Control input`，把接中控的那組 AF1062 控制輸入，綁定函式：

```text
General broadcast pattern start/stop（Level）  →  指向步驟 2 的樣式
```

!!! note "為什麼選「Level（準位）」而不是脈衝"
    **Level 的意思是「跟著接點狀態走」**：

    - 中控接點**閉合**（代表中控廣播中）→ CX 開始播該樣式
    - 中控接點**放開** → CX 停止

    所以請中控端把這個控制輸出設定成「**廣播期間持續閉合、結束才放開**」，CX 才能精準地與中控同起同停。若設成短脈衝，就無法對應中控實際的播放長度。

!!! warning "這條路徑是「一般廣播」，不是緊急"
    以上整套走的是**一般廣播（優先權 121–140）**，**不會進入緊急模式**，也不會封鎖 BGM 或搶下其他緊急廣播。它只是讓 CX 的區域跟著中控一起出聲而已。

## 二、外部線路音源不能當「緊急級」

這是整合時最容易誤會的地方，務必先講清楚。

!!! danger "緊急頻段只收「站台麥克風」與「內建預錄檔」"
    CX 的緊急頻段（優先權 1–80）只接受兩種音源：

    1. **站台麥克風**的 live 人聲（緊急麥克風廣播 41–60）
    2. SM1000 內的**內建預錄訊息**（緊急訊息 61–80）

    任何從外部進來的「線路音源」——包括中控 line out 接進 AF1062、或 RM1700 的 AUX 輸入——**最高只能到「一般 121」**，無法取得緊急優先權、無法 override BGM 與一般廣播。

換句話說：想讓外部中控的聲音「壓過一切、當緊急廣播用」是做不到的。若真的需要緊急級 override，正確做法是改用「外部接點觸發 CX 內建預錄緊急訊息」的架構（另見緊急廣播整合頁），而不是把中控的線路音源當緊急音源灌進來。

| 音源來源 | 可達最高優先權 | 能否當緊急 |
|---|---|---|
| 站台麥克風（OP/RM input1） | 41–60（緊急麥克風） | ✅ 可 |
| CX 內建預錄檔 | 61–80（緊急訊息） | ✅ 可 |
| 外部線路（中控 line、RM1700 AUX） | 121（一般） | ❌ 不可 |

## 三、CX 緊急要送到 IP-A1 喇叭：走類比橋接，不走網路

當現場的喇叭是 **IP-A1 系列**（走網路 multicast 收 CX 音訊）時，會遇到一個限制：**CX 的緊急廣播無法經由網路 multicast 送達 IP-A1**。

!!! danger "網路路徑不通"
    IP-A1 的 multicast 目的地只能掛在「Wide area（廣域）」型樣式上，而緊急廣播的目的地下拉**只吃 Normal / Individual 型**，不列 Wide area。因此「緊急走網路到 IP-A1」這條路走不通。

正確做法是**類比線路橋接**：

!!! tip "用 AF1062 線路輸出實體接進 IP-A1"
    1. 把某個 **AF1062 線路輸出**（例如命名為 `IP-A1 UI`）用音訊線**實體接到 IP-A1 系統的音訊輸入**。
    2. 在緊急樣式的 **Individual 目的地**下拉 `Broadcast audio output` 中，就會出現這個輸出，選它。
    3. IP-A1 側自行把這個 line-in 廣播出去（VOX/常開），並把優先權設得夠高。

    完整聲音路徑：

    ```text
    外部接點 → AF1062 控制輸入 → CX 緊急 pattern
        → Phase 目的地(Individual) = 該 AF1062 線路輸出
        → 類比線路 → IP-A1 系統 → IP-A1 喇叭
    ```

    這樣 IP-A1 喇叭聽到的，是 **CX 播出的預錄緊急訊息**（不是 IP-A1 自己的內建音源）。

!!! note "替代方案：IP-A1 各自並行觸發"
    也可以把緊急接點另外接到 IP-A1 的 SC15 控制輸入（僅 2 組）或走 HTTP Remote API，直接觸發 IP-A1 自己內建的緊急 pattern，與 CX 並行動作。三階段（警告/避難/解除）在 SC15 上可用 2 接點 + 1 組 HTTP 補足。

## 重點整理

- **同步跟播**：中控 line out → AF1062 音訊輸入（Usage=General）；中控開集極 → AF1062 控制輸入；於 `Broadcast → General broadcast pattern` 建樣式，再於 `Event activation → Control input` 綁「General broadcast pattern start/stop（**Level**）」。
- **Level = 跟著接點走**：中控接點閉合就播、放開就停；請中控把控制設成「廣播期間持續閉合」。
- **開集極極性**：AF「＋」→ 中控電晶體集極、AF「−」→ C 共通/GND，**接反不動作**；機械乾接點則不分極性。
- **同步跟播是「一般廣播（121–140）」**，不進緊急模式、不封鎖其他廣播。
- **外部線路音源上不了緊急**，最高只到一般 121；緊急只收站台麥克風與內建預錄檔。
- **CX 緊急送 IP-A1**：網路 multicast 不行，改用 **AF1062 線路輸出實體接 IP-A1 音訊輸入**，於 Individual 目的地下拉選它即可。
