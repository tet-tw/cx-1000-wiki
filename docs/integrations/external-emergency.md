# 外部緊急系統整合（3 接點）

用外部緊急系統的**警告 / 避難 / 解除**三組乾接點，接進 CX-1000 的 AF1062 音訊介面控制輸入，即可觸發 CX 內建的預錄緊急訊息，完成跨系統的緊急廣播連動。

!!! danger "危險：非消防認證，不可取代法定系統"
    CX-1000 屬於一般／緊急廣播設備，**並非 EN54 或消防認證產品**。本頁所述整合為「廣播連動」用途，**不得取代法規要求的火災偵測與疏散系統**。請務必保留合乎法規的獨立消防系統，本連動僅作為輔助廣播。

## 運作概念

外部緊急系統（例如既有的消防或災防主機）通常會在不同事件狀態下，提供幾組**乾接點輸出**。我們把其中三組接到 CX 的 AF1062 控制輸入，讓 CX 依接點狀態播放對應的預錄語音：

```text
外部緊急系統                AF1062 控制輸入        CX 動作
─────────────              ──────────────        ──────────────────────
[警告] 乾接點  ──────────►  CI-1               播放「警告」預錄訊息（進入緊急模式）
[避難] 乾接點  ──────────►  CI-2               緊急序列前進到「避難」階段
[解除] 乾接點  ──────────►  CI-3               播放「解除」訊息並退出緊急模式
```

!!! note "為什麼用內建預錄訊息，而不是把外部聲音直接送進來"
    CX 的緊急頻段（優先權 1–80）只接受「站台麥克風」與「內建預錄檔」兩種音源。外部系統的**線路（Line）音源進不了緊急頻段**，最高只能到一般廣播（優先權 121）。因此要做到真正壓過 BGM 與一般廣播的緊急 override，正解是：外部接點觸發 → CX 播放事先錄好的 Warning / Evacuation / All-clear。

## 事件對應表

三組接點各自對應到一個 CX 事件函式：

| 外部接點 | AF1062 控制輸入 | CX 事件函式 | 實際效果 |
|---|---|---|---|
| **警告** | CI-1 | Emergency message broadcast pattern **start** | 啟動緊急樣式，序列 Phase 1＝Warning 訊息循環播放 |
| **避難** | CI-2 | Emergency sequence **phase transition** | 序列前進到 Phase 2＝Evacuation 訊息 |
| **解除** | CI-3 | Emergency **all-clear** | 播放 All-clear、停止所有緊急訊息並退出緊急模式 |

!!! tip "階段推進的安全機制"
    「避難」階段是靠 CI-2 接點手動推進。建議在序列 Phase 1 把 Time-out 設為 ON、時長設一段足夠長的安全時間；萬一避難接點一直沒送來，超過安全時間後系統仍會自動前進到避難階段，避免卡在警告階段。

!!! warning "解除（All-clear）不放在序列裡"
    緊急序列的各 phase 只能選「Warning／Evacuation」用途的內容；**All-clear 不是序列的一環**。它掛在「Emergency all-clear」事件下，於復歸時才播放。所以「停止緊急廣播」的正確作法，就是觸發解除接點（CI-3）—— 一個動作同時完成：停掉緊急訊息、退出緊急模式、播出 All-clear。

## 接線重點

AF1062 共有 10 組控制輸入，三組接點綽綽有餘。電氣規格如下：

- 型式：**無電壓 make（常開）乾接點**
- 開放時端子電壓約 24V DC，短路電流約 5mA
- 負極（「−」）在 10 組輸入間**內部共通**

接法：每組外部接點，一腳接到對應的 **CI-1／CI-2／CI-3 的「＋」**，另一腳接到任一「−」（共通）。三組可共用同一條共通回線。

```text
外部「警告」接點  ── CI-1 (+)
外部「避難」接點  ── CI-2 (+)
外部「解除」接點  ── CI-3 (+)
三組另一腳       ── (−) 共通回線（可共用一條）
```

!!! danger "外部必須是無電壓（乾）接點"
    AF1062 控制輸入內部自帶光耦與 24V／5mA 供電，**外部只負責短路或開路，絕對不可送入帶電訊號**，否則會誤讀甚至損壞。若外部給的是帶電輸出，請中間加一顆隔離繼電器轉成乾接點再接入。若外部三組接點的回線必須彼此隔離，也是各加一顆隔離繼電器解決。

!!! warning "開集極／半導體輸出要注意極性"
    - **機械接點**（繼電器、按鈕）：不分極性，「＋」「−」任意接。
    - **半導體輸出**（開集極 OC／SSR）：**要分極性** —— AF「＋」接電晶體集極（高側）、AF「−」接射極／GND，接反不會動作。
    - 不想煩惱極性，最保險就是中間加隔離繼電器，把半導體輸出還原成機械乾接點。

!!! note "接點閉合時間"
    外部接點閉合需維持 **≥ 0.2 秒**，AF 才會判讀為有效觸發。若外部是極短瞬時脈衝，請延長或改用保持型輸出。

另外，請確認 **AF1062 的 24V DC 電源已接妥**（DC 電源輸入端子或 AD-246）。AF1062 端子上的 VCC／GND 兩條跳線維持出廠短接狀態即可，接乾接點不需更動。

## 事前準備（軟體側）

實際觸發前，工程設定端需先完成幾件事。這裡列出客戶／整合商需要知道的重點，詳細操作步驟另見施工文件：

1. **先開啟緊急模式**：`選單 → General → Emergency mode = Enable`。這一步要**最先做**，否則後面所有緊急相關的用途、優先權、事件函式欄位都不會出現；且事後改動 General 會重置相關設定。
2. **準備三個預錄語音檔**（Warning／Evacuation／All-clear）：
   - WAV／PCM、**16-bit、單聲道**、取樣率 16／32／48 kHz
   - 檔名 5–15 字元（含 `.wav`），僅限英數與 `! @ # $ % & - _`，**不可含空白**
   - 每檔 ≤ 250MB、全部合計 ≤ 500MB
   - **檔頭墊一小段靜音**，否則廣播開頭第一個字會被切掉
3. **事件綁定**：於 `選單 → Event activation → Control input`，把 AF1062 的 CI-1／CI-2／CI-3 分別綁上「事件對應表」中的三個事件函式。

!!! note "預設優先權（通常不用改）"
    - 避難（Evacuation）：61–80，預設 65
    - 警告（Warning）：61–80，預設 70
    - 解除（All-clear）：121–140，預設 125

    數字越小優先權越高，所以避難會壓過警告；解除屬一般頻段，播完即恢復正常。

## 測試驗收

上機後依序驗證三個接點：

1. 閉合**警告**接點 → 確認 Warning 訊息播放到指定區域、系統進入緊急模式。
2. 閉合**避難**接點 → 確認序列前進到 Evacuation 訊息。
3. 閉合**解除**接點 → 確認播出 All-clear、系統恢復正常。
4. 若有連動警示燈／繼電器一併確認，並實測外部接點閉合時間確實 ≥ 0.2 秒。

!!! warning "多 Hub 系統的解除範圍"
    AF1062 觸發的 All-clear 只會清除「它所屬的那個 hub」。若系統橫跨多個 hub，請確認負責解除的 AF1062 位在你要復歸的 hub 內。

## 重點整理

- 外部緊急系統的**警告／避難／解除**三組乾接點 → AF1062 控制輸入 **CI-1／CI-2／CI-3** → 觸發 CX 內建預錄緊急訊息。
- 事件對應：CI-1＝樣式啟動（Warning）、CI-2＝序列階段推進（Evacuation）、CI-3＝解除（All-clear，同時退出緊急模式）。
- 接線只認**無電壓乾接點**；機械接點不分極性，開集極／半導體輸出要對極性，必要時加隔離繼電器。
- 接點須閉合 **≥ 0.2 秒**才會被判讀。
- 軟體側**先開 Emergency mode**，並準備符合規格、檔頭墊靜音的三個 WAV 預錄檔。
- **本整合非 EN54／非消防認證，不得取代法規要求的消防偵測與疏散系統。**
